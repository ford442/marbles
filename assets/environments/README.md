# Environment Cubemaps

This directory contains KTX1 cubemap files used for specular IBL (Image-Based
Lighting) on high/ultra graphics quality.  Diffuse GI always uses the
hand-tuned 3-band spherical harmonics stored in `src/rendering/environment.js`.

## File pairs

| Preset        | IBL file                  | Skybox file                    |
|---------------|---------------------------|--------------------------------|
| default       | `default.ibl.ktx`         | `default.skybox.ktx`           |
| space_nebula  | `space_nebula.ibl.ktx`    | `space_nebula.skybox.ktx`      |
| ice           | `ice.ibl.ktx`             | `ice.skybox.ktx`               |
| volcanic      | `volcanic.ibl.ktx`        | `volcanic.skybox.ktx`          |
| neon_city     | `neon_city.ibl.ktx`       | `neon_city.skybox.ktx`         |

`*.ibl.ktx` — prefiltered specular cubemap (64 px base, 7 mip levels, 6 faces,
GL_RGB32F).  Loaded by `engine.createTextureFromKtx1()` and passed to
`IndirectLight.Builder().reflections()`.

`*.skybox.ktx` — single-mip skybox cubemap (64 px, 6 faces, GL_RGB32F).
Loaded by `engine.createTextureFromKtx1()` and passed to
`Skybox.Builder().environment()`.

## Synthetic vs photorealistic cubemaps

The files shipped here are **synthetic** — they are baked from the same SH
coefficients used for diffuse GI (generated by
`scripts/generate-env-cubemaps.js`).  Each mip level smoothly interpolates
between the full 3-band SH evaluation (sharpest) and the DC/ambient term
(softest), mimicking the behaviour of a prefiltered specular IBL without
requiring HDR photography.

For **photorealistic** reflections, replace any file with a cmgen-generated
output:

```sh
# Example: generate a 256-px prefiltered IBL from an .hdr equirect
cmgen --format=ktx --size=256 --compression=none \
      --output-format=float \
      --ibl-samples=1024 \
      -x assets/environments/ice \
      path/to/ice_glacier.hdr
# Rename the output to ice.ibl.ktx / ice.skybox.ktx
```

The runtime loading path (`loadIblCubemap` / `loadSkyboxCubemap` in
`src/rendering/environment.js`) is format-agnostic — any valid GL_RGB32F KTX1
file with 6 faces will work.  The hand-tuned SH is always used for diffuse GI
regardless of which cubemap is present.

## Format specification

| Field               | Value              |
|---------------------|--------------------|
| KTX version         | KTX 1              |
| glType              | `0x1406` GL_FLOAT  |
| glFormat            | `0x1907` GL_RGB    |
| glInternalFormat    | `0x8815` GL_RGB32F |
| Pixel dimensions    | 64 × 64 (base mip) |
| Number of faces     | 6                  |
| Mip levels (IBL)    | 7  (64 → 1 px)     |
| Mip levels (skybox) | 1  (64 px only)    |
| KVD "sh" metadata   | 27 space-separated floats (3-band SH, RGB triplets) |

Face order: +X, −X, +Y, −Y, +Z, −Z (standard cubemap order).
