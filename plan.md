# Hyphon Multi‑Agent Song Creation Guide

This document serves as the **definitive reference** for Hyphon's multi‑agent import pipeline and AISong workflow.  It grew out of the original plan and now covers everything from data formats to AI prompt templates, collaboration patterns, and version control strategies.

---
## 1. Overview

Hyphon is a browser‑based DAW that supports importing songs generated by multiple AI models and legacy hardware formats (RBS).  To keep the codebase scalable we operate with specialized agents:

* **AISongPhase2Agent** – handles Phase 2 enhancements in the AI‑song importer.
* **RbsExpansionAgent** – maintains RBS parsing and import logic.
* **PlanMasterAgent** – keeps this guide and associated documentation up to date.

Each agent works independently but follows conventions described below.

## 3. AISong workflow (Phase 2)

### 3.1 Feature list

* **Auto‑save** – every imported AISongData is written to `/songs/ai-generated/` with an ISO timestamp suffix.
  * Payload includes a `history` array capturing `sourceAI` and date.
* **Preview pane** – when the user selects a JSON file the modal renders a live preview.
* **Improved modal**
  * Progress bar indicating validation/import progress (0→100%).
  * Inline validation errors (wrong format, parse failures).
  * "Load Example Song" button to populate the preview pane with a stub.

### 3.2 API surface

```ts
// src/importers/ai-song/types.ts
export interface AISongData { ... }
export interface VersionHistoryEntry { ... }

// src/importers/ai-song/AiSongImporter.ts
export function saveAISongData(data: AISongData, sourceAI: string): string;
export function appendVersionHistory(filePath: string, sourceAI: string): void;
export function validateAISongData(data: any): boolean;
```

### 3.3 UI component

`ImportAiSongModal.tsx` contains the modal described above; it is intentionally framework‑agnostic so the host can integrate it into any view.

## 4. RBS importer enhancements

### 4.1 Data structures

`types.ts` now defines:

* `Slide`, `Accent`, `AutomationLane`, `PCFSettings` – covers expressive TB‑303 slides, accent strength, envelope automation and filter settings.
* `RbsPattern`, `RbsTrack`, `RbsSong` – high‑level song model used by the application.

### 4.2 Parsing improvements

`RbsParser.ts` reads a simple binary header (`version`, `type`, `length`) and stubs the loop where tracks/patterns are extracted.  Future work should implement the full spec.

### 4.3 Importer logic

`RbsImporter.ts` provides a convenience `importRbsFile(path)` that:

* Loads song via `RbsParser`.
* Converts 16‑step drums to a 32‑step internal representation using `DRUM_MAP_16_TO_32`.
* Applies a naive drum‑name mapping as an example.

## 5. Prompt templates

These templates were tuned for each model family; they assume the AI is given the high‑level goal of producing JSON conforming to `AISongData`.

```text
// Jules (long‑context creative model)
"Generate an AISongData object for a 4‑bar trance loop at 128bpm. Use nested tracks with melody and bass. Output pure JSON."

// Gemini (Google)
"You're a music assistant. Produce valid AISongData for a simple song. tempo=120, two tracks. No extra text."

// Claude
"Return JSON only. Schema:
{title:string,author:string,tempo:number,tracks:[{...}]}
Compose a short synthwave piece."

// Copilot
"/* AISongData schema: ... */
Write a JavaScript object named "song" that follows the schema and has a drum kick pattern."

// Grok
"Make an AISongData JSON with piano and drums. Give it a title and tempo."
```

Agents should stash final prompts in `prompt-templates.md` under the repo if they evolve.

## 6. Collaboration patterns

The multi‑agent pipeline can be decomposed into cooperating sub‑agents:

1. **Melody agent** – focuses on chord progressions and lead lines.
2. **Lyrics agent** – when vocals are required, generates textual lyrics to embed in metadata.
3. **Automation agent** – writes dynamic filter/volume automation lanes.

Coordination is achieved by passing partial `AISongData` objects between agents; the PlanMasterAgent documents hand‑off contracts.

---
## 7. Version control & HF Storage

We recommend using the `app_storage_manager.py` utility to push and fetch `/songs/ai-generated/` artefacts to HuggingFace Datasets.  The strategy:

1. Auto‑save locally on import.
2. On nightly CI run `python app_storage_manager.py sync` which commits new files to HF, preserving the original timestamp and `history` metadata.
3. Branches may contain experimental prompts; merge conflicts are resolved by renaming with `_vX` suffixes.
